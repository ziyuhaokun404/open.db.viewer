## 2. 原型目标

本结构稿用于定义 Open DB Viewer 首版的低保真产品原型框架，重点回答三个问题：

- 用户打开产品后先看到什么
- 用户如何完成“连接、浏览、查看、查询、导出”主流程
- 每个页面最小应该包含哪些模块与状态

这份文档不追求视觉风格设计，而是用于指导：

- 低保真线框图绘制
- 页面与模块拆分
- 交互流程梳理
- 后续技术模块映射

## 3. 原型设计原则

### 3.1 主流程优先

所有页面和模块优先服务以下核心链路：

1. 新建连接
2. 进入连接
3. 浏览对象树
4. 查看表结构
5. 查看表数据
6. 执行查询
7. 导出结果

### 3.2 简洁优先

避免一开始暴露过多专业功能。首版原型应让用户在视觉上快速理解：

- 左边是“数据库里有什么”
- 中间是“当前正在看什么”
- 顶部是“当前可以做什么”

### 3.3 信息层级清晰

界面结构应让用户快速区分：

- 连接层
- 对象层
- 内容层
- 结果层

### 3.4 默认安全

首版以浏览与查看为主，不应在主界面上强化高风险写操作。

## 4. 全局结构

建议采用桌面端三段式布局：

- 顶部：全局工具栏
- 左侧：连接与对象导航区
- 中央：主内容工作区
- 底部或右下：状态反馈区

建议基础布局如下：

```text
+--------------------------------------------------------------+
| 顶部工具栏：新建连接 | 刷新 | 查询 | 导出 | 连接状态 | 设置 |
+----------------------+---------------------------------------+
| 左侧导航区           | 中央主工作区                          |
| 连接列表             | 页面标题 / 面包屑                     |
| 数据库对象树         | 标签区：结构 / 数据 / 查询            |
| 搜索框（可选）       | 内容区域                              |
|                      | 底部状态栏 / 错误提示                 |
+----------------------+---------------------------------------+
```

## 5. 页面结构总览

首版建议包含 6 个核心页面或视图：

1. 启动页 / 连接主页
2. 新建连接页
3. 数据库浏览主界面
4. 表结构视图
5. 表数据视图
6. 查询视图

其中：

- `表结构视图 / 表数据视图 / 查询视图` 可作为数据库浏览主界面的内部页签
- `启动页 / 新建连接页` 可以作为独立页面或弹窗流程

## 6. 页面一：启动页 / 连接主页

### 6.1 页面目标

让首次用户一眼知道怎么开始，让老用户一眼看到已有连接。

### 6.2 页面结构

```text
+--------------------------------------------------------------+
| Logo / 产品名                                                |
| 一句说明：轻量数据库浏览工具                                  |
|                                                              |
| [ 新建连接 ]                                                 |
|                                                              |
| 最近连接 / 已保存连接                                         |
| ------------------------------------------------------------ |
| My Local MySQL        MySQL        上次使用时间              |
| Project PG            PostgreSQL   上次使用时间              |
| Local SQLite          SQLite       上次使用时间              |
| ------------------------------------------------------------ |
|                                                              |
| 空状态说明 / 使用提示                                         |
+--------------------------------------------------------------+
```

### 6.3 核心模块

- 产品标识区
- 主 CTA：`新建连接`
- 已保存连接列表
- 最近连接区
- 空状态提示

### 6.4 关键交互

- 点击 `新建连接` 进入连接创建流程
- 点击某个连接项直接进入数据库浏览主界面
- 鼠标悬浮连接项可见基础操作，如编辑、删除

### 6.5 状态设计

- 空状态：暂无连接，显示“新建连接”引导
- 常规状态：显示连接列表
- 错误状态：读取连接列表失败时显示重试入口

## 7. 页面二：新建连接页

### 7.1 页面目标

让用户以最小认知负担完成数据库连接创建与测试。

### 7.2 页面结构

```text
+--------------------------------------------------------------+
| 返回 | 新建连接                                               |
|                                                              |
| 数据库类型： [ MySQL ] [ PostgreSQL ] [ SQLite ]             |
|                                                              |
| 连接名称： [____________________]                            |
| 主机地址： [____________________]                            |
| 端口：     [________]                                        |
| 用户名：   [____________________]                            |
| 密码：     [____________________]                            |
| 数据库名： [____________________]                            |
|                                                              |
| SQLite 模式下：                                               |
| 数据库文件： [______________] [ 浏览 ]                       |
|                                                              |
| [ 测试连接 ]      [ 保存连接 ]                               |
|                                                              |
| 结果提示区：成功 / 失败 / 失败原因                           |
+--------------------------------------------------------------+
```

### 7.3 核心模块

- 数据库类型切换区
- 动态连接表单区
- 测试连接按钮
- 保存连接按钮
- 连接结果反馈区

### 7.4 字段设计建议

MySQL / PostgreSQL：

- 连接名称
- 主机地址
- 端口
- 用户名
- 密码
- 数据库名

SQLite：

- 连接名称
- 本地数据库文件路径

### 7.5 关键交互

- 切换数据库类型时动态切换表单字段
- 点击 `测试连接` 时显示加载状态
- 测试成功后强化 `保存连接`
- 保存成功后自动进入数据库浏览界面

### 7.6 状态设计

- 默认状态：可填写表单
- 测试中：按钮 loading，表单只读或部分禁用
- 测试成功：展示成功提示
- 测试失败：展示错误原因和修正提示

## 8. 页面三：数据库浏览主界面

### 8.1 页面目标

成为用户进入数据库后的主工作台，承载对象树浏览和内容切换。

### 8.2 页面结构

```text
+----------------------------------------------------------------------------------+
| 顶部工具栏：新建连接 | 刷新 | 查询 | 导出 | 当前连接名 | 连接状态 | 设置         |
+----------------------------+-----------------------------------------------------+
| 左侧导航区                 | 中央内容区                                          |
| 搜索框                     | 标题：数据库名 / 表名                               |
| [连接名]                   | 面包屑：连接 / 库 / Schema / 表                     |
|   - database_a             | -------------------------------------------------- |
|     - tables               | 页签：[结构] [数据] [查询]                          |
|       - users              |                                                     |
|       - orders             | 内容区域                                            |
|     - views                |                                                     |
|   - database_b             |                                                     |
|                            |                                                     |
+----------------------------+-----------------------------------------------------+
| 底部状态栏：加载状态 / SQL 状态 / 错误提示 / 结果条数                           |
+----------------------------------------------------------------------------------+
```

### 8.3 核心模块

- 顶部全局工具栏
- 左侧对象树
- 中央内容工作区
- 页签导航
- 底部状态栏

### 8.4 顶部工具栏建议

- `新建连接`
- `刷新`
- `查询`
- `导出`
- 当前连接信息
- 连接状态
- 设置入口

### 8.5 左侧导航区建议

- 对象搜索框
- 当前连接树
- 数据库 / Schema / 表 / 视图层级
- 当前选中对象高亮

### 8.6 主工作区建议

用户点击对象树里的某一张表后，中间区域默认打开该表，并展示：

- 页面标题
- 面包屑
- `结构 / 数据 / 查询` 三个页签

## 9. 页面四：表结构视图

### 9.1 页面目标

让用户快速理解表有哪些字段，以及字段的关键属性。

### 9.2 页面结构

```text
+----------------------------------------------------------------------------------+
| 表名：users                                                                       |
| 说明区：行数信息 / 更新时间 / 所属库信息                                           |
| -------------------------------------------------------------------------------- |
| 字段名        类型         可空        默认值        主键        备注              |
| id            bigint       否          -            是          用户ID             |
| email         varchar      否          -            否          邮箱               |
| created_at    timestamp    否          now()        否          创建时间           |
| -------------------------------------------------------------------------------- |
+----------------------------------------------------------------------------------+
```

### 9.3 核心模块

- 表基础信息区
- 字段列表表格
- 基础元信息展示区

### 9.4 关键交互

- 用户从对象树点击表后默认可切换到结构页
- 字段列表支持基础滚动查看
- 后续版本可扩展索引、外键等更多信息

### 9.5 最小展示字段

- 字段名
- 数据类型
- 是否可为空
- 默认值
- 是否主键

## 11. 页面六：查询视图

### 11.1 页面目标

为用户提供一个足够轻量但可用的查询区域，不走复杂 IDE 路线。

### 11.2 页面结构

```text
+----------------------------------------------------------------------------------+
| 查询对象：当前连接 / 当前数据库                                                    |
| 工具栏：执行 | 清空 | 导出结果                                                     |
| -------------------------------------------------------------------------------- |
| SQL 输入区                                                                      |
| SELECT *                                                                        |
| FROM users                                                                      |
| LIMIT 100;                                                                      |
| -------------------------------------------------------------------------------- |
| 结果区                                                                          |
| id   | email              | status                                               |
| 1    | a@test.com         | active                                               |
| 2    | b@test.com         | active                                               |
| -------------------------------------------------------------------------------- |
| 状态区：执行成功，耗时 120ms，返回 100 行                                         |
+----------------------------------------------------------------------------------+
```

### 11.3 核心模块

- 查询工具栏
- SQL 输入区
- 结果表格
- 状态与错误区

### 11.4 关键交互

- 点击执行运行当前 SQL
- 执行成功后展示结果
- 执行失败后展示错误提示
- 支持导出当前查询结果

### 11.5 首版边界

首版只强调：

- 单条查询执行
- 查询结果展示
- 基础错误提示

首版不强调：

- 智能补全
- 查询历史
- 多结果集
- 复杂 SQL 编辑体验

## 12. 关键弹层与辅助结构

除主页面外，建议补充以下辅助原型：

### 12.1 编辑连接弹窗

用途：

- 修改连接名称或连接参数

### 12.2 删除连接确认弹窗

用途：

- 防止误删已保存连接

建议文案：

- 标题：确认删除连接？
- 内容：删除后不会影响数据库本身，只会移除本地保存的连接配置

### 12.3 错误提示弹层

用途：

- 展示连接失败、查询失败、数据加载失败等错误

### 12.4 导出成功提示

用途：

- 提醒用户导出完成和文件保存位置

## 13. 全局状态与反馈设计

首版原型中建议统一以下反馈方式：

### 13.1 Loading

适用场景：

- 测试连接
- 加载对象树
- 加载表结构
- 加载表数据
- 执行查询

设计建议：

- 按钮级 loading
- 内容区 loading
- 状态栏显示当前动作

### 13.2 Empty

适用场景：

- 没有保存连接
- 对象树无内容
- 表数据为空
- 查询无结果

### 13.3 Error

适用场景：

- 连接失败
- 元数据读取失败
- 查询执行失败
- 导出失败

设计建议：

- 用简洁语句说明失败原因
- 给出可恢复动作，如重试、返回编辑连接

### 13.4 Success

适用场景：

- 连接测试成功
- 连接保存成功
- 导出成功

## 14. 页面跳转关系

建议主流程跳转如下：

```text
启动页
  -> 新建连接页
  -> 测试连接成功
  -> 保存连接

      -> 表数据视图
      -> 查询视图

  -> 点击已有连接
  -> 数据库浏览主界面
```

## 16. 原型结论

Open DB Viewer 首版原型不需要追求复杂，而要追求：

- 首屏就知道怎么开始
- 主界面就知道库表在哪里
- 打开表就知道结构和数据怎么看
- 想查数据时有一个足够顺手的查询入口

从原型结构上看，最关键的是把下面这条路径画顺：

- 进入产品
- 新建连接
- 进入数据库
- 找到表
- 看结构
- 看数据
- 查结果
- 导出

只要这个路径在原型上足够清晰，后续无论是 UI 设计还是研发实现，都会更容易落地。
